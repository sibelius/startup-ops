How full text search works ?

Full text search

  • For each new or updated item we update the inverted index (elastic search)
  • We do a full text search using elastic search, and get the whole document from the primary database

Architecture of Webhooks

Webhooks flow

  • Flagship notify of a new transaction
  • Server receives the requests - flagship sends the webhook to us
  • Server saves the transaction to the database and emits an event of new transaction
  • Worker process the event
  • Send a request to each webhook registered for this event
  • Try 7 times using exponential backoff

Webhook Reliability

  • How to make sure we won't miss any webhook ?

Webhook Reliability

  • Uses a state machine to track if the charge/transaction webhook was already sent
  • If the charge/transaction didn't notify the webhook
  • The cron fallback job will retry to send the webhook request
